Detailed Description

Dislocation trajectory analysis module.

This module contains functions that are used in the script dislocation_script.py.

The functions contained in this module allow the user to analyse Molecular Dynamics (MD) trajectories of straight edge dislocations generated using LAMMPS.

The script ovito_disloc.py post-processes dislocation trajectories generated as LAMMPS dumps files containing the unwrapped coordinates of atoms. The functions in this script use as input the text files generated by ovito_disloc.py, which uses the OVITO Dislocation Analysis (DXA) tool to detect dislocations. The format of the text files generated by ovito_disloc.py is described in further detail in its documentation.

The function track_disloc included here allows the user to track dislocations across the various frames of a trajectory, since in a simulation containing multiple dislocations, different dislocations are indexed differently by the DXA tool. Accordingly, this function allows the user to "track" a given dislocation throughout a trajectory for further processing such as studying its dynamics.

The script dislocation_script.py shows how the other functions in this module can be used to carry out additional processing of dislocation data files that is necessary prior to using track_disloc.

Track dislocation lines across multiple frames.
- Dislocations are tracked by checking the minimum distance moved by a dislocation between frames. 

- Adapted from the approach in:
https://pysource.com/2021/10/05/object-tracking-from-scratch-opencv-and-python/

- Assumptions:
1. The dislocation only moves in the x direction
2. The number of dislocations is constant across all frames 
i.e. no dislocations are generated nor lost throughout the analysed trajectory.

234 2. The number of dislocations is constant across all frames

236

239 avg : numpy.ndarray
240 Average x, y, and z coordinates of dislocation lines in the analysed trajectory.
241 x_lim : float
242 Simulation cell length in x (Angstroms).

244 Number of frames in trajectory.

367 # Sort vertices along y coords

370 frames = []

381
382 # Unwrap x to be consistent with average position data
383 for i, start in enumerate(starts):
384
385 for s in start:
386 for j in range(s,len(pbcs[i])):
387 for k in range(len(disloc[i][j])):
388 disloc[i][j][k][0] += x_lim
389
